# 第20回 量子特徴マップ（Feature Map）の設計戦略

## はじめに

前回は：

* 勾配ベース最適化
* Parameter Shift Rule
* COBYLA
* SPSA
* Adam
* Layer-wise training
* ノイズ対応戦略

を扱いました。

ここまでで：

変分量子回路を

安定して学習させる方法

が整理できました。

しかし：

量子機械学習の性能を決める

もう一つの重要要素があります。

それが：

**特徴マップ（Feature Map）**

です。

今回は：

分類性能を左右する

特徴マップ設計の考え方

を解説します。

---

# Feature Mapとは何か？

特徴マップとは：

古典データを量子状態に変換する操作

です。

つまり：

```text
x → |ψ(x)⟩
```

という変換です。

量子機械学習では：

この変換が

モデル性能の中心

になります。

---

# なぜFeature Mapが重要なのか？

理由：

量子回路は

入力された状態しか処理できない

からです。

つまり：

入力の質

＝

学習性能

になります。

これは：

古典機械学習と同じです。

---

# 古典機械学習との対応関係

古典機械学習：

特徴量設計が性能を決める

量子機械学習：

Feature Map設計が性能を決める

つまり：

最も重要な設計対象の一つです。

---

# 最も単純なFeature Map

最小構成：

```python
qc.ry(x, 0)
```

これは：

1次元入力を

回転角として使う方法です。

直感：

値がそのまま状態に反映されます。

---

# 多次元データの埋め込み

2次元データ：

```python
qc.ry(x1, 0)
qc.ry(x2, 1)
```

つまり：

各量子ビットに

特徴を割り当てます。

これは：

最も基本的な方法です。

---

# 問題点：線形すぎる

単純な回転だけでは：

線形変換

しか作れません。

結果：

複雑な分類問題に弱い

という欠点があります。

---

# エンタングルメントを追加する

改善方法：

量子ビット間の相互作用を追加します。

例：

```python
qc.ry(x1, 0)
qc.ry(x2, 1)
qc.cx(0, 1)
```

これで：

非線形特徴空間

が作られます。

---

# ZZFeatureMapとは何か？

代表的なFeature Map：

ZZFeatureMap

です。

例：

```python
from qiskit.circuit.library import ZZFeatureMap

feature_map = ZZFeatureMap(feature_dimension=2, reps=2)
```

特徴：

エンタングルメントを含む
非線形変換が可能
実績が多い

量子カーネルでもよく使われます。

---

# なぜZZFeatureMapが強いのか？

理由：

特徴量同士の相互作用

を表現できるからです。

つまり：

```text
x1 × x2
```

のような関係を

自然に表現できます。

これは：

古典MLの多項式特徴

に対応します。

---

# repsパラメータの意味

例：

```python
ZZFeatureMap(feature_dimension=2, reps=3)
```

意味：

特徴変換の繰り返し回数

です。

repsが増えるほど：

非線形性が強くなります。

---

# ただしrepsを増やしすぎると？

問題：

回路が深くなる
ノイズが増える
最適化が難しくなる

つまり：

適度な値

が重要です。

---

# データ再アップロードとは何か？

高度なFeature Map設計：

**Data Re-uploading**

という方法があります。

例：

```python
qc.ry(x, 0)
qc.rz(x, 0)
qc.ry(x, 0)
```

特徴：

同じデータを複数回使う

これにより：

表現能力が向上します。

---

# なぜData Re-uploadingが効くのか？

理由：

量子回路の深さが増えるほど

非線形性が強くなる

からです。

つまり：

少ない量子ビットでも

高表現力が得られます。

---

# 回転軸の選び方

Feature Mapでは：

回転軸も重要です。

例：

```python
RX(x)
RY(x)
RZ(x)
```

違い：

RX：X方向回転
RY：Y方向回転
RZ：位相回転

これらを組み合わせると：

表現能力が向上します。

---

# 位相情報の重要性

量子計算では：

位相

が重要な情報になります。

例：

```python
qc.rz(x, 0)
```

これは：

確率を変えずに

状態を変化させます。

つまり：

干渉構造を調整できます。

---

# Feature Map設計の基本戦略

実践的な設計手順：

① 各特徴を量子ビットへ割り当てる
② 回転ゲートで埋め込む
③ エンタングルメントを追加
④ 必要なら繰り返す

これが基本構造です。

---

# 問題依存Feature Mapとは何か？

より高度な方法：

問題構造に合わせて設計する

例：

時間データ：

周期回転を使う

グラフ構造：

接続関係をCXで表現

物理問題：

ハミルトニアン構造を反映

つまり：

意味のある埋め込み

が可能になります。

---

# Feature MapとExpressibilityの関係

重要な関係：

Feature Mapが弱い

→ 学習できない

Feature Mapが強すぎる

→ 最適化が難しい

つまり：

バランス

が必要です。

---

# Feature MapとKernel法の関係

量子カーネルでは：

Feature Mapが

そのまま性能を決めます。

つまり：

```text
K(x, y) = ⟨ψ(x)|ψ(y)⟩
```

の構造は

Feature Map次第です。

---

# VQCとFeature Mapの関係

VQCでは：

Feature Map

＋

Ansatz

の組み合わせで性能が決まります。

つまり：

両方が重要です。

---

# 実務でのおすすめ構成

まず試す構成：

ZZFeatureMap
＋
RealAmplitudes

これは：

最も安定した組み合わせです。

---

# Feature Map設計のチェックポイント

設計時の確認項目：

非線形性があるか
エンタングルメントがあるか
回路が深すぎないか
問題構造を反映しているか

これが重要です。

---

Feature Mapとは：

量子機械学習における

特徴量設計そのもの

です。

つまり：

モデル性能の鍵

になります。
