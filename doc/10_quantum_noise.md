# 第10回 量子ノイズとは何か？

## はじめに

前回は：

* QAOA（量子近似最適化アルゴリズム）
* コストハミルトニアン
* ミキサーハミルトニアン
* 変分最適化
* p層拡張

を扱いました。

ここまでで：

VQE
QAOA

という

**NISQ時代の代表アルゴリズム**

を実装できるようになりました。

しかし：

これらはすべて

理想的なシミュレータ上

で動かしていました。

現実の量子コンピュータでは：

**ノイズ**

が存在します。

今回は：

量子ノイズとは何か？

を理解していきます。

---

# なぜ量子ノイズが重要なのか？

現在の量子コンピュータは：

完全ではありません。

特徴：

* 状態が壊れる
* ゲートが不正確
* 測定が誤る

つまり：

計算結果が乱れます。

これが：

NISQ（Noisy Intermediate-Scale Quantum）

という名前の由来です。

---

# ノイズとは何か？

量子ノイズとは：

量子状態が意図しない方向に変化すること

です。

原因：

* 環境との相互作用
* ハードウェア誤差
* 制御誤差
* 測定誤差

などがあります。

---

# 量子状態は壊れやすい

古典ビット：

0 または 1

量子ビット：

a|0⟩ + b|1⟩

つまり：

量子状態は

連続値

です。

そのため：

少しの誤差でも

状態が変化します。

---

# 主な量子ノイズの種類

代表的なノイズは：

3種類あります。

* デコヒーレンス
* ゲート誤差
* 測定誤差

順番に見ていきます。

---

# デコヒーレンスとは何か？

デコヒーレンスとは：

量子状態が環境と相互作用して

重ね合わせが壊れる現象

です。

例えば：

0.707|0⟩ + 0.707|1⟩

が

|0⟩

または

|1⟩

に崩れます。

つまり：

量子性が失われます。

---

# T1緩和とは何か？

T1とは：

エネルギー緩和時間

です。

意味：

|1⟩ が |0⟩ に戻る時間

です。

例えば：

励起状態が自然に崩壊する

現象です。

---

# T2緩和とは何か？

T2とは：

位相緩和時間

です。

意味：

位相情報が失われる時間

です。

例：

0.707|0⟩ + 0.707|1⟩

が

ランダム位相になる

現象です。

---

# なぜT2が重要なのか？

量子計算では：

位相が情報

だからです。

つまり：

T2が短いほど

量子アルゴリズムは難しくなります。

---

# ゲート誤差とは何か？

量子ゲートは：

理想通りには動きません。

例えば：

理想：

RY(π/2)

実際：

RY(π/2 + ε)

になります。

つまり：

少しずつ誤差が蓄積します。

---

# 測定誤差とは何か？

測定では：

|0⟩ を

|1⟩

と誤認識する場合があります。

例：

実際：

|0⟩

測定結果：

|1⟩

これが：

測定誤差

です。

---

# ノイズはなぜ問題なのか？

量子回路が深くなるほど：

誤差が増えます。

つまり：

ゲート数が多いほど

成功確率が下がります。

そのため：

NISQアルゴリズムでは

浅い回路

が重要になります。

---

# ノイズの影響を観察する

簡単な例として：

Hadamard回路を考えます。

理想状態：

50% / 50%

でした。

しかし：

ノイズがあると：

60% / 40%

のように偏ります。

---

# シミュレータでもノイズを扱える

Qiskitでは：

ノイズモデル

を追加できます。

例：

```python
from qiskit_aer.noise import NoiseModel
```

これにより：

実機に近い計算

が可能になります。

---

# なぜノイズシミュレーションが重要なのか？

理由：

実機は

すぐに使えない場合が多い

からです。

そのため：

まず

ノイズ付きシミュレーション

で検証します。

---

# ノイズと回路設計の関係

ノイズがある環境では：

回路設計が重要になります。

例えば：

* ゲート数を減らす
* 深さを減らす
* 測定回数を増やす

などの工夫を行います。

---

# エラー耐性とは何か？

量子アルゴリズムでは：

ノイズに強い構造

が重要です。

例：

VQE
QAOA

は：

ノイズに比較的強い

アルゴリズムです。

これが：

NISQ向き

と呼ばれる理由です。

---

# ノイズと誤り訂正の関係

将来の量子コンピュータでは：

量子誤り訂正

が使われます。

例：

surface code

しかし：

現在はまだ

実用段階ではありません。

そのため：

ノイズ前提で設計する

必要があります。

---

# まとめ

今回は：

* デコヒーレンス
* T1緩和
* T2緩和
* ゲート誤差
* 測定誤差
* ノイズ付き計算

を扱いました。

ここまでで：

理想量子回路

から

現実の量子計算

への橋渡しができました。

次回は：

ノイズシミュレーション

を実際に実装しながら：

実機に近い量子回路実行

を体験していきます。
