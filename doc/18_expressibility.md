# 第18回 量子回路の表現能力（Expressibility）とは何か？

## はじめに

前回は：

* 量子回帰モデル
* 期待値による連続値出力
* MSE損失関数
* EstimatorQNN
* NeuralNetworkRegressor

を扱いました。

ここまでで：

量子回路を

分類器
回帰モデル

として使えるようになりました。

しかし：

ここで重要な疑問が生まれます。

**どんな量子回路でも学習できるのか？**

答えは：

**いいえ**

です。

今回は：

量子回路の性能を決める重要な概念：

**Expressibility（表現能力）**

を解説します。

---

# Expressibilityとは何か？

Expressibilityとは：

量子回路がどれだけ多様な量子状態を表現できるか

を表す指標です。

つまり：

回路の「表現力」

です。

ニューラルネットで言えば：

層の深さ
ノード数

に相当します。

---

# なぜExpressibilityが重要なのか？

理由：

表現能力が低い回路は

どれだけ学習しても

良いモデルにならない

からです。

つまり：

最適化以前に

設計の問題

になります。

---

# 例：表現能力が低い回路

例：

```python
qc.ry(theta, 0)
```

この回路は：

1つの回転しかできません。

つまり：

表現できる状態は

限定されます。

その結果：

複雑な関数は近似できません。

---

# 表現能力が高い回路

例：

```python
qc.ry(theta1, 0)
qc.rz(theta2, 0)
qc.rx(theta3, 0)
```

さらに：

エンタングルメントを追加：

```python
qc.cx(0, 1)
```

これで：

表現可能な状態空間

が大きくなります。

---

# 直感的な理解

表現能力とは：

探索できる状態空間の広さ

です。

狭い回路：

単純な関数しか表現できない

広い回路：

複雑な関数を表現できる

という違いがあります。

---

# ニューラルネットとの対応関係

比較：

ニューラルネット：

層が深いほど表現能力が高い

量子回路：

ゲートが多いほど表現能力が高い

つまり：

回路深さ

が重要になります。

---

# しかし深い回路には問題がある

ここで問題が発生します：

深い回路

＝

ノイズに弱い

です。

つまり：

表現能力を上げたい

しかし：

回路は浅くしたい

という

トレードオフ

があります。

---

# エンタングルメントの役割

表現能力を高める最大の要素：

エンタングルメント

です。

理由：

量子ビット同士が独立ではなくなる

からです。

例：

独立状態：

```text
|ψ⟩ = |a⟩⊗|b⟩
```

エンタングル状態：

```text
|ψ⟩ ≠ |a⟩⊗|b⟩
```

つまり：

より複雑な状態を表現できます。

---

# repsパラメータの意味

RealAmplitudesなどの回路には：

reps

があります。

例：

```python
RealAmplitudes(num_qubits=2, reps=3)
```

意味：

回路の繰り返し回数

です。

repsが増えるほど：

表現能力が向上します。

---

# しかしrepsを増やしすぎると？

問題：

ノイズが増える

最適化が難しくなる

学習が不安定になる

つまり：

適切な深さ

が重要です。

---

# barren plateau問題とは何か？

Expressibilityを考えるとき：

重要な問題があります：

**barren plateau**

です。

これは：

勾配が消える現象

です。

つまり：

学習できなくなります。

---

# barren plateauの直感

イメージ：

損失関数が

ほぼ平坦

になります。

結果：

どの方向に更新すればよいか

分からなくなります。

つまり：

最適化が停止します。

---

# なぜbarren plateauが起きるのか？

原因：

回路が複雑すぎる

ランダムすぎる

エンタングルメントが多すぎる

です。

つまり：

表現能力が高すぎても問題

になります。

---

# 良い回路設計とは何か？

理想：

十分な表現能力

＋

学習可能な構造

です。

つまり：

バランス

が重要です。

---

# よく使われるansatz

代表例：

RealAmplitudes
EfficientSU2
HardwareEfficientAnsatz

これらは：

実用的な表現能力

を持つよう設計されています。

---

# Hardware Efficient Ansatzとは何か？

特徴：

ハードウェア構造に合わせた回路

です。

メリット：

浅い回路で実装可能

つまり：

実機向き

です。

---

# 問題依存ansatzとは何か？

別のアプローチ：

問題構造に合わせて回路を設計

します。

例：

化学問題：

UCC ansatz

最適化問題：

QAOA ansatz

つまり：

用途特化型回路

です。

---

# Expressibilityの測定方法

研究分野では：

Expressibilityは

数値的に評価されます。

例：

Haarランダム状態との距離

などを使います。

ただし：

開発者レベルでは：

そこまで必要ありません。

重要なのは：

設計の直感

です。

---

# 実務での設計指針

回路設計の基本：

量子ビット数を増やす
エンタングルメントを入れる
repsを調整する

この3つです。

---

# Expressibilityと最適化の関係

重要な関係：

表現能力が低い

→ 学習できない

表現能力が高すぎる

→ 勾配が消える

つまり：

中間が最適

です。

---

# 開発者にとっての意味

Expressibilityとは：

回路設計の品質指標

です。

つまり：

アルゴリズム性能を決める

重要要素です。
