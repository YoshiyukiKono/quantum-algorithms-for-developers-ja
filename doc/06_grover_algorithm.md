## はじめに

前回は：

* 干渉

を扱いました。

そこで：

量子コンピュータは
複数の入力を同時に処理できる

今回は：

**Grover探索アルゴリズム**

を実装します。

これは：

量子計算による高速化

を最も直感的に理解できるアルゴリズムです。

---

# 探索問題とは何か？

次の問題を考えます：

4つの候補の中から
正解を1つ探す

例：

| 候補 |
| -- |
| 00 |
| 01 |
| 10 |
| 11 |

この中のどれか1つが正解です。

---

# 古典コンピュータの場合

古典コンピュータでは：

最悪4回

調べる必要があります。

平均でも：

約2回

必要です。

---

# 量子コンピュータの場合

Groverアルゴリズムでは：

約 √N 回

で解けます。

今回の例では：

√4 = 2

つまり：

2回未満の操作

で解けます。

これが：

量子探索高速化

です。

---

# Groverアルゴリズムの流れ

アルゴリズムは：

次の4ステップです。

1. 重ね合わせを作る
2. 正解状態の位相を反転する
3. 平均について反転する
4. 測定する

この：

位相反転 + 平均反転

の組み合わせが核心です。

---

# 回路の準備

まず：

2量子ビット回路を作ります。

コード：

```python id="s7n1fq"
from qiskit import QuantumCircuit

qc = QuantumCircuit(2, 2)
```

---

# 重ね合わせを作る

すべての候補を同時に表現します。

```python
qc.h(0)
qc.h(1)
```

状態：

0.5(|00⟩ + |01⟩ + |10⟩ + |11⟩)

になります。

---

# オラクルとは何か？

Groverでは：

正解状態だけ

符号を反転します。

例えば：

|11⟩ を正解とすると：

|11⟩ → −|11⟩

に変換します。

---

# オラクルを実装する

今回は：

|11⟩ を反転します。

コード：

```python
qc.cz(0, 1)
```

これで：

|11⟩ の位相だけが反転します。

---

# 平均について反転する（Diffusion演算子）

次に：

平均について反転する操作

を行います。

これは：

Diffusion演算子

と呼ばれます。

コード：

```python
qc.h(0)
qc.h(1)

qc.x(0)
qc.x(1)

qc.h(1)
qc.cx(0, 1)
qc.h(1)

qc.x(0)
qc.x(1)

qc.h(0)
qc.h(1)
```

これが：

Groverアルゴリズムの核心部分です。

---

# 測定を追加する

最後に：

測定を追加します。

```python
qc.measure([0, 1], [0, 1])
```

---

# 全体コード

まとめると：

```python id="kg4nrm"
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.compiler import transpile

qc = QuantumCircuit(2, 2)

qc.h(0)
qc.h(1)

qc.cz(0, 1)

qc.h(0)
qc.h(1)

qc.x(0)
qc.x(1)

qc.h(1)
qc.cx(0, 1)
qc.h(1)

qc.x(0)
qc.x(1)

qc.h(0)
qc.h(1)

qc.measure([0, 1], [0, 1])

simulator = AerSimulator()

compiled = transpile(qc, simulator)

job = simulator.run(compiled, shots=1000)

result = job.result()

print(result.get_counts())
```

---

# 実行結果

結果：

{'11': 1000}

つまり：

正解だけが出力されます。

これは：

干渉によって

正解状態の確率だけが増幅された

ことを意味します。

---

# なぜ確率が増えるのか？

Groverでは：

次の操作を行っています：

正解状態の符号を反転
↓
平均について反転

これによって：

正解状態の確率振幅が増加します。

つまり：

干渉を利用した増幅

です。

---

# Grover反復とは何か？

Groverアルゴリズムでは：

この操作を

複数回繰り返します。

これを：

Grover反復

と呼びます。

反復回数は：

約 √N 回

です。

---

# なぜ高速化できるのか？

古典探索：

1つずつ調べる

量子探索：

全候補を同時に扱う
＋
干渉で正解だけ強調する

これが：

量子探索の本質です。

---

# Groverアルゴリズムの応用

Groverは：

様々な探索問題に応用できます。

例：

* データベース探索
* パスワード探索
* 最適化問題
* SAT問題

つまり：

探索問題全般

に適用可能です。

---

# Deutsch–Jozsaとの違い

Deutsch–Jozsa：

1回で判定できる特殊問題

Grover：

一般探索問題を高速化

つまり：

実用性があるのは

Grover

です。
