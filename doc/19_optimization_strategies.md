# 第19回 量子回路の最適化戦略

## はじめに

前回は：

* Expressibility（表現能力）
* 回路深さと性能の関係
* エンタングルメントの役割
* barren plateau問題
* ansatz設計の考え方

を扱いました。

そこから分かった重要な点は：

量子アルゴリズムの性能は

**最適化の成否に大きく依存する**

ということです。

今回は：

量子回路を安定して学習させるための

**最適化戦略**

を整理します。

---

# なぜ最適化が難しいのか？

量子機械学習では：

次の問題が発生します：

* ノイズがある
* 勾配が消える
* 測定誤差がある
* 評価関数が確率的

つまり：

古典ニューラルネットより

最適化が難しい

です。

---

# 量子最適化の基本構造

変分量子アルゴリズムでは：

次のループを繰り返します：

1 パラメータ設定
2 回路実行
3 測定
4 損失関数計算
5 パラメータ更新

この：

④ → ⑤

の部分が

最適化です。

---

# 最適化アルゴリズムの種類

量子回路では：

主に3種類の最適化手法が使われます：

① 勾配ベース最適化
② 勾配なし最適化
③ 確率的最適化

順番に見ていきます。

---

# 勾配ベース最適化とは？

例：

Gradient Descent

です。

更新式：

```
θ ← θ − η ∂L/∂θ
```

メリット：

収束が速い

デメリット：

ノイズに弱い

---

# Parameter Shift Ruleとは何か？

量子回路では：

通常の微分が使えません。

代わりに：

Parameter Shift Rule

を使います。

例：

```
∂f(θ)/∂θ = f(θ + π/2) − f(θ − π/2)
```

つまり：

2回の回路実行で

勾配を計算できます。

---

# 勾配なし最適化とは？

代表例：

COBYLA
Nelder-Mead

特徴：

勾配を使わない

つまり：

ノイズに強い

です。

そのため：

NISQ時代では

最もよく使われます。

---

# COBYLAとは何か？

COBYLAは：

制約付き最適化アルゴリズム

です。

特徴：

少ない評価回数
ノイズ耐性
安定した収束

VQCやVQEで

よく使われます。

---

# SPSAとは何か？

SPSA：

Simultaneous Perturbation Stochastic Approximation

です。

特徴：

少ない回路評価回数

で更新できます。

通常の勾配：

2n回評価

SPSA：

2回評価

で済みます。

---

# なぜSPSAが重要なのか？

理由：

量子回路は

実行コストが高い

からです。

つまり：

評価回数削減

が重要になります。

---

# Adamは使えるのか？

はい、使えます。

Adamは：

確率的勾配法の改良版

です。

特徴：

高速収束
局所最適に強い

ただし：

ノイズの影響を受けやすい

という欠点があります。

---

# 最適化アルゴリズムの選び方

基本方針：

小規模回路：

Gradient Descent

中規模回路：

Adam

実機実行：

COBYLA / SPSA

が適しています。

---

# 初期値の重要性

量子回路では：

初期値が非常に重要です。

理由：

barren plateau

を避けるためです。

例：

悪い初期値：

```
θ = random large value
```

良い初期値：

```
θ ≈ 0
```

---

# Layer-wise trainingとは何か？

Layer-wise trainingとは：

回路を段階的に学習する方法

です。

手順：

1 reps=1で学習
2 reps=2へ拡張
3 reps=3へ拡張

こうすると：

安定します。

---

# 局所コスト関数を使う

別の戦略：

局所コスト関数

を使う方法です。

例：

全量子ビットではなく

一部だけ測定する

メリット：

勾配消失を防げる

---

# 正則化とは何か？

正則化とは：

過学習を防ぐ方法

です。

例：

```
loss = original_loss + λ||θ||
```

量子回路でも：

同様に使えます。

---

# 学習率の調整

学習率（learning rate）

も重要です。

大きすぎる：

発散

小さすぎる：

収束が遅い

つまり：

適切な調整

が必要です。

---

# ノイズを考慮した最適化

実機では：

ノイズが存在します。

そのため：

次の工夫が有効です：

shots数を増やす
平均化する
浅い回路を使う

---

# バッチ学習は可能か？

可能です。

方法：

データを分割して

順番に学習します。

これは：

ミニバッチ学習

と呼ばれます。

量子機械学習でも

有効です。

---

# ハイブリッド最適化とは何か？

ハイブリッド最適化とは：

複数の最適化手法を組み合わせる方法

です。

例：

前半：

COBYLA

後半：

Adam

これで：

安定性と速度

を両立できます。

---

# 実務でのおすすめ戦略

実践的な順序：

① 浅い回路
② COBYLA開始
③ SPSA切替
④ Layer-wise拡張

これが：

最も安定します。

---

# 最適化とExpressibilityの関係

重要な関係：

表現能力が高い

→ 最適化が難しい

表現能力が低い

→ 学習できない

つまり：

設計と最適化はセット

です。

---

# 開発者にとっての意味

量子アルゴリズムでは：

回路設計

と同じくらい

最適化設計

が重要です。

つまり：

最適化は

アルゴリズムの一部

です。
