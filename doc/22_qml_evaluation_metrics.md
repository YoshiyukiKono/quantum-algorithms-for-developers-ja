# 第22回 量子機械学習モデルの評価指標

## はじめに

前回は：

* 量子カーネル法とVQCの違い
* ノイズ耐性の比較
* スケーラビリティ
* 実装コスト
* モデル選択戦略

を整理しました。

ここまでで：

量子機械学習モデルを

どのように選ぶか

が分かりました。

しかし：

もう一つ重要な問題があります。

**モデル性能をどう評価するのか？**

今回は：

量子機械学習モデルの評価指標

を整理します。

---

# なぜ評価指標が重要なのか？

量子機械学習では：

精度が高いだけでは

十分ではありません。

理由：

* ノイズがある
* 回路コストがある
* 測定回数が必要
* 実機制約がある

つまり：

複数の観点で評価する必要があります。

---

# 評価指標の3つの軸

量子機械学習では：

次の3つを評価します：

① 予測性能
② 計算コスト
③ 実機適合性

順番に見ていきます。

---

# 予測性能とは何か？

まず：

通常の機械学習と同じ指標です。

分類問題：

Accuracy
Precision
Recall
F1-score

回帰問題：

MSE
MAE
R²

これらは：

そのまま使えます。

---

# Accuracyとは何か？

Accuracy：

正解率

です。

定義：

```text id="g2f8v8"
Accuracy = 正解数 / 全データ数
```

最も基本的な指標です。

---

# Precisionとは何か？

Precision：

予測の信頼性

です。

定義：

```text id="sl0cbm"
Precision = TP / (TP + FP)
```

誤検出を評価します。

---

# Recallとは何か？

Recall：

検出能力

です。

定義：

```text id="h2mdv9"
Recall = TP / (TP + FN)
```

見逃しを評価します。

---

# F1-scoreとは何か？

F1-score：

PrecisionとRecallの平均

です。

定義：

```text id="v7wzsy"
F1 = 2 × Precision × Recall / (Precision + Recall)
```

分類問題でよく使われます。

---

# 回帰問題の評価指標

代表例：

MSE：

```text id="d8g9be"
MSE = 平均二乗誤差
```

MAE：

```text id="38mnlm"
MAE = 平均絶対誤差
```

R²：

```text id="psw4ys"
決定係数
```

です。

---

# しかし量子機械学習ではこれだけでは不十分

理由：

量子回路には

追加コスト

があるからです。

例えば：

shots数
回路深さ
量子ビット数

などです。

---

# 回路深さとは何か？

回路深さ：

量子ゲートの段数

です。

例：

```text id="q6q42k"
浅い回路 → ノイズに強い
深い回路 → 表現能力が高い
```

つまり：

重要な評価指標です。

---

# 量子ビット数とは何か？

量子ビット数：

使用する量子資源

です。

少ないほど：

実機で実行しやすい

というメリットがあります。

---

# shots数とは何か？

shots数：

測定回数

です。

例：

```text id="3hij3h"
shots = 1024
```

shotsが多いほど：

精度が高い

ですが：

計算時間が増えます。

---

# Expressibilityとの関係

モデル評価では：

Expressibility

も重要です。

理由：

表現能力が低いと

学習できない

からです。

---

# Trainabilityとは何か？

Trainability：

学習しやすさ

です。

評価対象：

勾配の大きさ
収束速度
安定性

などがあります。

---

# barren plateauの検出

評価方法の一つ：

勾配の分散

を測定します。

もし：

分散が小さい

なら：

barren plateau

の可能性があります。

---

# Generalizationとは何か？

Generalization：

未知データへの適応能力

です。

つまり：

過学習していないか

を確認します。

---

# 交差検証とは何か？

代表的な方法：

Cross Validation

です。

例：

```text id="nprhka"
train / test 分割
```

または：

k-fold validation

を使います。

---

# ノイズ耐性の評価

量子モデルでは：

ノイズ耐性

も評価対象です。

方法：

ノイズモデル追加
FakeBackend使用
実機実行比較

などがあります。

---

# 実機適合性とは何か？

重要な指標：

実機で動くか？

です。

評価ポイント：

回路深さ
量子ビット数
接続制約

です。

---

# 実機向け評価指標

追加指標：

transpile後の回路深さ
CNOT数
実行時間
エラー率

などがあります。

---

# Quantum Advantageの評価

研究分野では：

Quantum Advantage

も評価対象になります。

つまり：

古典モデルより優れているか？

です。

ただし：

現在は限定的です。

---

# 実務での評価フロー

実践的な評価手順：

① Accuracy測定
② 回路深さ確認
③ shots数確認
④ ノイズ耐性評価
⑤ 実機実行確認

これが基本です。

---

# モデル比較のチェックリスト

比較時の観点：

精度
回路深さ
量子ビット数
shots数
学習時間
ノイズ耐性

これで判断できます。

---

# 開発者にとっての意味

量子機械学習では：

精度だけではなく

量子資源

も評価対象になります。

つまり：

資源効率

が重要です。
